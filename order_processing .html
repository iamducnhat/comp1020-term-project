2Interface Definition — OrderQueue<T>

OrderQueue.java

interface 
/**
 * OrderQueue — generic interface for all queue types.
 * Supports both FIFO (regular) and priority-based (VIP).
 *
 * T09 – Phase 3 Design | VinTony POS Loyalty System
 */
public interface OrderQueue {

    /**
     * Add an order to the queue.
     * Regular: adds to tail (FIFO)
     * VIP: inserts by loyaltyPoints (max-heap)
     */
    void enqueue(Order order);

    /**
     * Remove and return the highest-priority order.
     * Regular: removes from head (FIFO)
     * VIP: removes order with most loyaltyPoints
     *
     * @throws NoSuchElementException if queue is empty
     */
    Order dequeue();

    /**
     * Return the next order WITHOUT removing it.
     * Useful for display / preview logic.
     *
     * @throws NoSuchElementException if queue is empty
     */
    Order peek();

    /** Number of orders currently waiting. */
    int size();

    /** True if no orders are waiting. */
    boolean isEmpty();
}

RegularOrderQueue.java

FIFO 
/**
 * Regular FIFO queue — backed by LinkedList.
 * First order in = first order out.
 */
public class RegularOrderQueue
        implements OrderQueue {

    private final LinkedList<Order> queue
            = new LinkedList<>();

    @Override
    public void enqueue(Order order) {
        queue.addLast(order);   // O(1)
    }

    @Override
    public Order dequeue() {
        if (isEmpty())
            throw new NoSuchElementException();
        return queue.removeFirst();  // O(1)
    }

    @Override
    public Order peek() {
        if (isEmpty())
            throw new NoSuchElementException();
        return queue.peekFirst();
    }

    @Override
    public int size()    { return queue.size(); }
    @Override
    public boolean isEmpty() { return queue.isEmpty(); }
}

VIPOrderQueue.java

PriorityQueue 
/**
 * VIP queue — backed by java.util.PriorityQueue.
 * Orders with MORE loyaltyPoints are dequeued first.
 * Tie-break: earlier orderTime wins (FIFO within tier).
 */
public class VIPOrderQueue
        implements OrderQueue {

    // max-heap: higher loyalty = higher priority
    private final PriorityQueue<Order> queue =
        new PriorityQueue<>(
            Comparator
              .comparingInt((Order o) ->
                  o.getCustomer().getLoyaltyPoints())
              .reversed()
              .thenComparing(Order::getOrderTime)
        );

    @Override
    public void enqueue(Order order) {
        queue.offer(order);   // O(log n)
    }

    @Override
    public Order dequeue() {
        if (isEmpty())
            throw new NoSuchElementException();
        return queue.poll();   // O(log n)
    }

    @Override
    public Order peek() {
        if (isEmpty())
            throw new NoSuchElementException();
        return queue.peek();
    }

    @Override
    public int size()    { return queue.size(); }
    @Override
    public boolean isEmpty() { return queue.isEmpty(); }
}

OrderController.java (excerpt)

controller 
public class OrderController {

    private final OrderQueue vipQueue
            = new VIPOrderQueue();
    private final OrderQueue regularQueue
            = new RegularOrderQueue();

    /** Route incoming order to correct queue. */
    public void placeOrder(Order order) {
        if (order.isVip())
            vipQueue.enqueue(order);
        else
            regularQueue.enqueue(order);
    }

    /**
     * Dequeue next order for processing.
     * VIP queue is always drained first.
     */
    public Order processNext() {
        if (!vipQueue.isEmpty())
            return vipQueue.dequeue();
        if (!regularQueue.isEmpty())
            return regularQueue.dequeue();
        throw new NoSuchElementException(
            "No pending orders");
    }

    public int getQueueSize() {
        return vipQueue.size()
             + regularQueue.size();
    }
}

3Time Complexity

Operation Method RegularOrderQueue (LinkedList) VIPOrderQueue (PriorityQueue)
Insert order enqueue() O(1) O(log n)
Remove next dequeue() O(1) O(log n)
View next peek() O(1) O(1)
Check empty isEmpty() O(1) O(1)
Queue size size() O(1) O(1)
Space — O(n) O(n)
Tại sao dùng PriorityQueue cho VIP?
java.util.PriorityQueue implement min-heap. Ta dùng .reversed() trên comparator để biến thành max-heap — order của customer có loyaltyPoints cao hơn sẽ được dequeue trước, dù đến sau trong cùng VIP segment. Tie-break bằng orderTime đảm bảo fairness khi hai customer cùng điểm.

4Design Decisions

Pattern
Interface-first
OrderQueue là interface. Controller chỉ biết interface — dễ swap implementation sau này (e.g. Redis queue khi scale).
Priority rule
Loyalty points
VIP priority dựa trên Customer.loyaltyPoints — tích hợp tự nhiên với loyalty system. Customer platinum (nhiều điểm hơn) được phục vụ trước gold.
Fairness
VIP trước, FIFO sau
processNext() drain hết VIP queue trước. Regular customers không bị block vô hạn — VIP queue rỗng thì regular queue chạy ngay.
